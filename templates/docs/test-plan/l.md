## Test Data Strategy（测试数据策略）

<!-- 资金/一致性测试数据尤其要明确：造什么前置、怎么造、怎么清理、怎么隔离 -->

- **前置数据**：本计划测试需要的前置（已付订单/激活设备/合同状态/分润配置…），逐项列
- **造法**：API 造 / SQL 直造 / 夹具；每类数据的造法 + 落库表
- **清理**：per-spec afterAll / 共享数据 reset；规则（禁止污染存量、禁止跨 spec 残留）
- **隔离**：独立数据 vs 共享；并发 workers 策略（共享数据须 workers=1）；租户隔离
- **资金/一致性特例**：金额用分(BIGINT)；对账差异数据怎么造（应 vs 实的偏差注入点）；幂等测试的重复投递注入法
